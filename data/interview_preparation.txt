# AI应用开发类岗位面试准备总纲

## 1. 文档用途
本知识文档用于 AI 应用开发类岗位的面试准备。

适用岗位包括：
- AI Agent 应用开发工程师
- 大模型工程师
- RAG / 知识库工程师

本文件的作用是帮助候选人从以下几个方面准备面试：
- 岗位能力模型
- 常考方向
- 准备重点
- 回答策略
- 项目表达方式

---

## 2. 面试准备的总体原则

### 2.1 先理解岗位，再准备知识
面试准备不要只背题，要先理解岗位职责。

建议先弄清楚：
- 这个岗位主要做什么
- 这个岗位解决什么业务问题
- 这个岗位需要什么技术栈
- 更看重工程能力还是研究能力
- 是否强调落地、稳定性、效率和协作

### 2.2 以项目经验为核心
AI 应用开发类岗位通常更看重：
- 项目经验
- 技术理解
- 表达能力
- 工程落地能力

面试时要尽量结合自己的项目经验来回答，而不是只讲概念。

### 2.3 回答问题要结构化
建议使用下面的回答结构：
1. 先给结论
2. 再讲原理或思路
3. 再结合项目或场景举例
4. 最后补充优缺点和改进方向

---

## 3. AI Agent 应用开发工程师面试准备

### 3.1 岗位核心能力
AI Agent 应用开发工程师通常关注以下能力：
- 大模型调用与集成能力
- Agent 架构设计能力
- 工具调用能力
- 任务拆解能力
- 多轮对话与上下文管理能力
- Prompt 设计能力
- 任务编排与流程控制能力
- 工程落地与调试能力

### 3.2 常考方向
#### Agent 基础概念
需要理解：
- 什么是 Agent
- Agent 与普通聊天机器人有什么区别
- 什么是 ReAct 思想
- 什么是任务规划、工具调用、执行反馈
- Agent 适合解决什么问题，不适合解决什么问题

#### 工具调用设计
需要掌握：
- 工具（Tool）的定义方式
- 工具输入输出如何设计
- 如何让模型决定调用哪个工具
- 如何处理工具调用失败和异常
- 如何让 Agent 多步执行任务

#### 多轮对话与上下文管理
需要理解：
- 如何保存上下文
- 如何控制上下文长度
- 如何保持任务目标一致
- 如何避免模型忘记前文

#### Prompt 与流程设计
需要掌握：
- 系统提示词如何设计
- 如何引导模型输出结构化结果
- 如何减少幻觉
- 如何设计任务型 Prompt 模板
- 如何让模型稳定调用工具

#### 工程能力
需要具备：
- 调试 Agent 行为的能力
- 处理错误和异常的能力
- 日志与可观测性的意识
- 重试机制的意识
- 对响应时间和成本的基本认知

### 3.3 面试准备重点
建议重点准备：
- 一个完整的 Agent 项目案例
- 一个工具调用或多步任务案例
- 一个你优化过 Prompt 的案例
- 一个你解决过 Agent 不稳定问题的案例

### 3.4 回答建议
回答 Agent 相关问题时，可以重点说明：
- 为什么要用 Agent
- Agent 的目标是什么
- 采用了什么架构
- 工具如何设计
- 如何保证执行稳定
- 如何评估效果

## 4. 大模型工程师面试准备

### 4.1 岗位核心能力
大模型工程师通常关注：
- 大模型调用
- 模型部署
- Prompt 设计
- 推理优化
- 输出结构化与可控性
- 系统集成
- 成本控制
- 稳定性保障

### 4.2 常考方向
#### 大模型基础理解
需要了解：
- Transformer 基本原理
- 预训练与微调的区别
- 指令微调、对齐、RLHF 的基本概念
- 大模型的能力边界与局限

#### 模型调用与部署
需要掌握：
- 如何调用模型 API
- 如何设计模型服务接口
- 如何控制超时、重试、限流
- 如何评估模型响应质量
- 如何管理模型版本

#### 推理优化
需要理解：
- 流式输出
- 缓存策略
- Token 控制
- 响应时间优化
- 成本与性能平衡

#### Prompt 工程
需要掌握：
- 如何设计高质量 Prompt
- 如何稳定输出 JSON、列表、步骤
- 如何通过示例和约束减少歧义
- 如何复用 Prompt 模板

#### 工程集成
需要理解：
- 大模型如何与业务系统连接
- 如何和检索、工具调用、数据库、消息队列协作
- 如何设计更稳定的链路

### 4.3 面试准备重点
建议重点准备：
- 模型调用和服务化经验
- Prompt 优化案例
- 性能优化和成本控制案例
- 输出不稳定时的调试经验

### 4.4 回答建议
回答大模型工程师问题时，可以突出：
- 你是否理解模型能力边界
- 你是否能把模型能力工程化
- 你是否能控制输出格式和稳定性
- 你是否关注性能、成本和可观测性

## 5. RAG / 知识库工程师面试准备

### 5.1 岗位核心能力
RAG / 知识库工程师通常关注：
- 文档处理
- 文档切分
- 向量化
- 检索
- 检索质量评估
- 知识库构建与维护
- 检索与生成结合
- 相关性判断
- 兜底策略

### 5.2 常考方向
#### RAG 基础
需要理解：
- 什么是 RAG
- RAG 解决了什么问题
- RAG 的基本流程
- 检索和生成各自的作用

#### 知识库构建
需要掌握：
- 文档如何解析
- 如何切分 chunk
- chunk 大小和重叠如何影响效果
- 如何做去重和更新
- 如何维护知识库内容质量

#### 检索策略
需要理解：
- 向量检索的基本思路
- 如何做 Top-K 召回
- 如何判断检索结果是否相关
- 如何避免无关内容干扰生成

#### RAG 评估与优化
需要掌握：
- 检索命中率
- 回答相关性
- 回答完整性
- 幻觉控制
- 检索不足时的兜底策略

#### 工程实践
需要理解：
- 如何组织知识库目录
- 如何做增量更新
- 如何做日志和监控
- 如何处理重复入库

### 5.3 面试准备重点
建议重点准备：
- 如何构建一个知识库
- 如何调优 chunk、top-k 和召回效果
- 如何判断一次检索是否有效
- 如何处理“检索不到”或“检索不相关”的情况

### 5.4 回答建议
回答 RAG 相关问题时，建议说明：
- 知识库来源是什么
- 文档如何处理
- 检索如何做
- 生成如何结合检索结果
- 如何减少幻觉
- 如何做 fallback

## 6. 通用面试准备知识

### 6.1 自我介绍
自我介绍建议包含：
1. 基本背景
2. 技术方向
3. 项目经验
4. 与岗位匹配点
5. 为什么想投这个岗位

#### 自我介绍原则
- 时间控制在 1 到 3 分钟
- 突出与岗位相关的内容
- 不要过度罗列经历
- 要体现“我适合这个岗位”

### 6.2 面试着装
面试着装建议：
- 整洁
- 简洁
- 干净
- 偏正式或商务休闲风格

建议避免：
- 过于夸张
- 过于随意
- 过于花哨

### 6.3 行为举止
面试时建议注意：
- 说话清晰
- 语速适中
- 保持眼神交流
- 回答前可以稍微思考
- 不要频繁打断面试官
- 不懂时可以诚实说明

### 6.4 回答问题的技巧
推荐回答结构：
- 结论先行
- 原理或思路说明
- 项目或案例举例
- 优缺点分析
- 总结提升

常见加分点：
- 能结合项目讲
- 能说清楚取舍
- 能说明为什么这么设计
- 能体现工程思维
- 能说明后续优化方向

### 6.5 行为面试题准备
常见行为题包括：
- 你做过最有挑战性的项目是什么
- 你如何处理冲突
- 你如何与团队协作
- 你遇到过什么失败和困难
- 你如何解决问题

推荐使用 STAR 法则：
- S：情境
- T：任务
- A：行动
- R：结果

### 6.6 面试中常见错误
- 只讲概念，不讲项目
- 只讲结果，不讲过程
- 回答过于发散
- 不懂时硬答
- 过度背题
- 不理解岗位需求就盲目准备

---

## 7. 面试准备建议总结
面试准备的核心不是背诵，而是建立“岗位能力模型”：
- 这个岗位要做什么
- 需要什么技能
- 常考什么问题
- 我有哪些项目可以支撑
- 我能否清楚表达自己的思路

对于 AI 应用开发类岗位，最重要的是：
- 理解大模型能力边界
- 理解 RAG、Agent、Prompt 的基本原理
- 具备工程落地和调试能力
- 能用项目说明自己的能力
- 能在面试中清晰表达设计思路和取舍
